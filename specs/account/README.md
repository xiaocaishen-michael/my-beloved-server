# Account Module

> `mbw-account` — 用户身份 / 凭证 / 会话 / 注销状态机。M1.1 第一个业务模块。

## Module identity (4-place naming, MUST match)

| Aspect | Value |
|--------|-------|
| Maven module | `mbw-account` |
| Java package | `com.mbw.account` |
| DB schema | `account` |
| Frontend feature | `features/account` (in `no-vain-years-app`) |

ArchUnit + Spring Modulith Verifier enforce these at CI time. See [meta repo `docs/conventions/business-naming.md`](https://github.com/xiaocaishen-michael/no-vain-years/blob/main/docs/conventions/business-naming.md).

## Domain model preview

(Materializes as PR-3+ use cases land via SDD four-step workflow. Below is the planned target shape.)

```text
Account (聚合根)
  ├── id: AccountId (BIGINT, autogenerated)
  ├── status: AccountStatus (ACTIVE / FROZEN / ANONYMIZED)
  ├── credentials: Set<Credential>
  │     ├── PhonePasswordCredential (P)
  │     ├── GoogleCredential        (G)
  │     └── WeChatCredential        (W)
  ├── profile: AccountProfile (display name, avatar, ...)
  └── createdAt / lastLoginAt (UTC)
```

- State machine: see [`account-state-machine.md`](./account-state-machine.md)
- Password policy: see [`password-policy.md`](./password-policy.md)
- Full PRD: see [meta repo `docs/requirement/account-center.v2.md`](https://github.com/xiaocaishen-michael/no-vain-years/blob/main/docs/requirement/account-center.v2.md)

## Cross-module dependencies

- **mbw-shared**: error codes, base utilities (currently empty; populated as needed)
- **mbw-billing** (M1.3+): inject `mbw-billing.api.EntitlementApi` for quota checks
- **mbw-pkm** (M2+): emit `AccountAnonymizedEvent` to trigger PKM data cascade-cleanup (Spring Modulith event + outbox)

No business module may directly import another business module's non-`api` package — enforced by Spring Modulith Verifier.
